# Audit 2026-06-29 - PRD 2.8 Dashboard Direction vs Current Implementation

## Scope

This is a read-only audit of the current implementation against the updated active docs:

- `AGENTS.md`
- `PRD.MD`
- `DESIGN.md`
- `PROJECT_STATUS.md`

Rules for this audit:

- `AGENTS.md` and `PRD.MD` are treated as the latest product and workflow source of truth.
- `DESIGN.md` is treated as the current visual reference only, not final dashboard layout approval.
- `PROJECT_STATUS.md` is implementation context only.
- No code was modified during the audit.

## Summary

The updated PRD changes the Dashboard direction significantly.

The new model is:

```txt
Dashboard = Inbox + Notes + Pulse
Daily = full checklist
Projects = full project database
```

The current implementation still follows the previous Dashboard model:

```txt
Dashboard = Hunting Overview + Today Focus + Active Hunts + Account Workload
```

This means the current Dashboard preview is now outdated relative to `PRD.MD` 2.8, even though it still matches much of the older `DESIGN.md` dashboard section.

## 1. What Currently Matches The Updated PRD

### Auth

Current auth mostly matches the updated PRD and AGENTS rules:

- Login uses username + password.
- Signup uses username, display name, password, and password confirmation.
- No user-facing email field exists.
- No email verification UI exists.
- Internal Supabase email is generated as `{username}@web3-hunting.local`.
- Internal email is not shown in the UI.
- Username validation uses `^[a-z0-9_]{3,24}$`.
- Auth errors are safe and do not expose provider internals.

### Development Auth Bypass

Current bypass still follows the allowed shape:

- Only active when `NODE_ENV === "development"`.
- Only active when Supabase env vars are missing.
- Has a clear console warning.
- Production should fail closed through `getSupabaseEnv()`.
- Does not run if Supabase env vars exist.

Small concern:

- The bypass condition is duplicated in multiple route files. This is not a new product conflict, but it can drift later.

### Daily

Current Daily preview matches the stable Daily direction:

- Daily owns the full checklist.
- Daily is grouped by account.
- Each account has Checklist and Running/Recheck.
- Running/Recheck are not checkboxes.

### Projects

Current Projects preview still matches the Projects direction:

- Projects is table-first.
- Desktop uses a full-width table.
- Hunt categories are tabs inside Projects.
- Sidebar does not contain hunt categories.
- Mobile has compact project cards.

### Visual Style

Current UI generally matches the broad visual direction:

- Dark productivity workbench.
- Soft-depth shell.
- Royal-charcoal tokens are already partly applied.
- Icons are Lucide.
- No terminal green base surfaces.
- No crypto news widget.
- No Gmail integration.
- No AI/discovery implementation.

## 5. Whether Dashboard, Inbox, Notes, Daily, And Projects Direction Is Clear

### Dashboard

Product direction is clear in PRD:

```txt
Dashboard = Inbox + Notes + Pulse
```

Dashboard should not be:

- mini Daily page
- mini Projects page
- analytics dashboard
- crypto news dashboard

Current code does not match this yet.

### Inbox

Direction is clear:

- Phase 1 Inbox is manual-first.
- Inbox is raw input.
- It is not email-only.
- Manual and quick capture sources are Phase 1.
- Gmail, Twitter/X, browser extension, API sources are future.
- No automatic conversion without user confirmation.

### Notes

Direction is clear:

- Global Notes are Phase 1.
- Project Notes remain inside project detail.
- Pinned Notes and Recent Notes may appear on Dashboard.
- Notes can optionally link to project/task/account/wallet/inbox item.

### Daily

Direction is clear:

- Full checklist lives in Daily.
- Grouped by account by default.
- Running/Recheck split remains.

### Projects

Direction is clear:

- Projects owns the full project database.
- Table-first, desktop-first.
- Hunt categories stay inside Projects as filters/tabs.
