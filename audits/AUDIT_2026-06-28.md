# Audit 2026-06-28 - Updated Docs vs Current Implementation

## Scope

This is a read-only audit of the current implementation against the updated active root docs:

- `AGENTS.md`
- `PRD.MD`
- `DESIGN.md`

`PROJECT_STATUS.md` was used only for current implementation context. The active source of truth remains the root docs above.

No code was changed during this audit.

## What Currently Matches

### Dashboard

- Dashboard no longer duplicates the full Daily checklist.
- `Today focus` is a compact signal list.
- `Active hunts` exists as a mini table.
- `Account workload` exists and includes only Moree, Wdym, and Wayss.
- `Running & recheck` exists as a separate monitoring section.
- `Recent activity` exists as a short preview.
- No crypto news, alpha feed, or discovery widget is shown.
- No Phase 2 or Phase 3 dashboard feature appears to have slipped in.

### Visual Style

- The app uses dark surfaces, raised panels, soft shadows, and inset controls.
- Lucide icons are used consistently.
- Base surfaces are not green or teal.
- Semantic colors are mostly limited to status, warning, success, danger, info, and progress.
- The app shell already has a rounded desktop-frame direction.

### Daily

- Daily remains the owner of the full checklist.
- Daily is grouped by account by default.
- Daily has Checklist plus Running/Recheck sections.
- Dashboard does not show checkbox-heavy task rows anymore.

### Projects

- Projects remains table-first on desktop.
- Hunt categories are tabs inside Projects, not sidebar navigation.
- Projects shows more than 12 realistic preview rows.
- Mobile fallback exists as compact project cards.

### Auth

- Login UI uses username + password.
- Signup UI uses username, display name, password, and password confirmation.
- No email field is shown.
- No verify-email page or email confirmation flow is present in the UI.
- Internal email is generated server-side as `{username}@web3-hunting.local`.
- Internal email is not displayed to the user.
- Auth errors are safe and do not expose provider internals.

### Development Auth Bypass

The development bypass looks mostly safe:

- Only active when `NODE_ENV === "development"`.
- Only active when Supabase environment variables are missing.
- Console warning is clear.
- Production with missing env vars should fail closed through `getSupabaseEnv()`.
- Middleware does not bypass if Supabase env vars exist.

## Conflicts With Updated Docs

### Missing Hunting Overview Strip

Updated `PRD.MD` and `DESIGN.md` require a Hunting Overview strip near the top:

```txt
Header / context
Quick Capture
Hunting Overview strip
Today Focus
Active Hunts
Account Workload
Running / Recheck
Upcoming Deadlines
Recent Activity
```

Current dashboard has four stat cards:

```txt
Active projects
Daily workload
Running / recheck
Completed today
```

This does not match the required Hunting Overview strip.

Required metrics include:

```txt
Total Projects
Active
Free Hunts
Retro
NFT
Waitlist
Trading
Archived
```

### Dashboard Order Mismatch

Current order:

```txt
Header
Quick capture
4 stat cards
Active hunts
Today focus
Account workload
Running / recheck
Recent activity
```

Updated docs want:

```txt
Header
Quick capture
Hunting Overview
Today Focus
Active Hunts
Account Workload
Running / Recheck
Upcoming Deadlines
Recent Activity
```

Main mismatch:

- `Today Focus` should appear before `Active Hunts`.
- `Upcoming Deadlines` is missing.
- The stat cards should become Hunting Overview.

### Quick Capture Missing Wallet Text

Updated PRD says quick capture should support:

```txt
Capture task, project, wallet, or note
```

Current quick capture says:

```txt
Capture a task, project, or note...
```

The wallet mention is missing. The visible buttons are still Task, Project, Note, which matches the docs, but the capture copy should include wallet if we follow PRD exactly.

### Visual Tokens Not Fully Updated

Current `globals.css` still uses earlier darker tokens:

```txt
--background: #111315
--card: #1b1d20
--secondary: #121416
--depth-stage: #3a3a38
```

Updated `DESIGN.md` wants a richer royal-charcoal system:

```txt
Stage Top #34373A
Stage Bottom #1E2023
Shell #16181B
Canvas #1E1F22
Surface #24262A
Surface Elevated #2B2E33
Surface Raised #31353A
```

Current style is close, but not fully aligned with DESIGN v1.2.

### Stage and Shell Ambience Not Fully Implemented

Updated `DESIGN.md` recommends subtle atmospheric gradients for:

- outer stage
- app shell ambient lighting
- restrained header ambience

Current app shell uses a flat stage color, plain background, shadow, rounded border. It does not yet implement the recommended stage or shell gradient treatment.

### Some UI Still Feels Admin or Devtool

Current risks:

- Stat cards still feel like generic dashboard KPI cards.
- Projects table has a fairly hard grid and sticky border treatment.
- Some controls and panels feel functional but not yet premium.
- Dashboard lacks the explicit `Hunting Overview` identity, so the top area still feels generic.

### Upcoming Deadlines Missing

Updated docs explicitly include:

```txt
Upcoming Deadlines
```

Current dashboard has no dedicated deadline block.

### Development Auth Bypass Logic Is Duplicated

The dev bypass condition appears in:

- `src/lib/supabase/middleware.ts`
- `src/app/page.tsx`
- `src/app/daily/page.tsx`
- `src/app/projects/page.tsx`

It is currently safe enough, but duplicated logic can drift later. Better to centralize it during auth cleanup.
